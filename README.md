# Verifis

Verifis is being revamped into a native macOS desktop app for very fast text explanation and fact-checking. The old Chrome extension and API are still in the repo, but the new primary app lives in `macos/`.

## macOS App

- Runs as a background menu-bar app with global hotkeys.
- `Control-Shift-T`: analyze highlighted text.
- `Control-Shift-S`: drag a screen area and analyze the text in it.
- Results appear as a compact frosted-glass toast anchored to the draggable pill.
- Ollama Local is the default no-key provider, with `llama3.2:3b` selected by default for a fast local baseline. `gpt-oss:20b` is available as an OpenAI open-weight local option.
- Gemini, OpenAI, and Anthropic are also supported with latency-oriented editable defaults: `gemini-3.1-flash-lite`, `gpt-5.4-nano`, and `claude-3-5-haiku-20241022`.
- API keys are stored in the macOS Keychain.
- ChatGPT subscriptions do not include OpenAI API usage, so Verifis does not automate or scrape chatgpt.com as a backend.
- Gemini can also use OAuth through Google Cloud Application Default Credentials.
- OpenAI and Anthropic direct model APIs require API keys, so the app keeps those providers on API-key auth.
- The app logo is generated by `script/generate_app_icon.swift` and exported to `macos/Sources/Resources/AppIcon.png`, `AppIcon.icns`, the iconset, and extension icon sizes.

### Run

```bash
./script/build_and_run.sh
```

The Codex Run action is wired to the same script through `.codex/environments/environment.toml`.

### No-Key Local AI

Verifis can run without API keys through Ollama:

```bash
brew install --cask ollama
open -a Ollama
ollama pull llama3.2:3b
```

For OpenAI's larger open-weight local model, use `ollama pull gpt-oss:20b` and select it in Verifis settings. Then choose `Ollama Local`. The default server is `http://127.0.0.1:11434`.

### macOS Permissions

- Accessibility lets Verifis copy highlighted text from other apps.
- Screen Recording lets Verifis OCR whole-screen and selected-area text.
- Network access is required for Gemini, OpenAI, or Anthropic analysis. Ollama Local talks only to the local Ollama server.

### Gemini OAuth

Gemini OAuth uses Google Cloud ADC. Enable the Google Generative Language API for your project, then run:

```bash
gcloud auth application-default login --scopes='https://www.googleapis.com/auth/cloud-platform,https://www.googleapis.com/auth/generative-language.retriever'
```

In Verifis, choose `Gemini`, set `Auth` to `OAuth`, and optionally enter the Google Cloud project ID for the `x-goog-user-project` header.

## Chrome Extension

## Features

- **Two Modes**:
  - **Fact Check**: Extracts factual claims from text and finds relevant sources
  - **Explain**: Provides simplified summaries and background context for complex text
- **Automatic Processing**: Highlight text on any webpage and get instant results
- **AI-Powered**: Uses Google Gemini 2.0 Flash for analysis
- **Smart Debouncing**: Waits for stable text selection before processing

## Setup

### Prerequisites

- Node.js 18+
- Chrome browser
- Google Gemini API key (get one at https://aistudio.google.com/app/apikey)

### Installation

1. Install dependencies and build:
```bash
cd extension
npm install
npm run build
```

2. Load the extension in Chrome:
   - Open `chrome://extensions/`
   - Enable "Developer mode"
   - Click "Load unpacked"
   - Select the `extension/dist` directory

3. Configure API key:
   - Click the extension icon
   - Click the settings gear icon
   - Enter your Gemini API key
   - Save

## Usage

1. Navigate to any webpage
2. Select the mode (Fact Check or Explain) in the popup
3. Highlight text on the page
4. Wait ~1 second for processing
5. View results in the extension popup

### Fact Check Mode
- Extracts 1-3 factual claims from selected text
- Finds up to 5 sources per claim with stance indicators (supports/contradicts/unclear)

### Explain Mode
- Provides a simplified summary in plain language
- Adds contextual background information
